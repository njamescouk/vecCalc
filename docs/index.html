1 Introduction

Having recovered from the outrageous success of simplePathWriter, which I’ve probably used about three times in the last five years, I grew frustrated with it’s limitations. Having tried and failed to do geometry with it I trawled around and found some remarkable software for doing the necessary. However these packages involved learning curves and some pretty idiosyncratic UI misfeatures. One, which really produces the goods, has a theorem prover attached, which is way over my pay grade.

So here we are: yet another not invented here lash up! This one is lightweight, unsuprising and programmer friendly. It produces .svg files of simple (no curves) geometrical figures from an equally simple text specification. This means the figures are easily and accurately editable and reproducible.

Enjoy!

2 Installation

Download the zip, extract to somewhere handy, use vecCalc.exe. c++ source code is included, it should compile on any respectable operating system.

3 Synopsis

vecCalc -h? -v? -d? -o? -svg? filename?

options are:

-d          : dump coordinates
-h          : this help
-o          : output filename
-svg        : (default)write svg paths and circles
-v          : version

The optional filename argument contains the specification for the figure.

4 Action

The intuition is that vecCalc is used in the same way you would go about setting a figure out. Define a couple of points, draw a line between them, bisect the line, draw a perpendicular and so on and so forth.

vecCalc reads the input, consisting of instructions to place points and lines in the plane, along with rendering directions for line style, visibility etc etc. the svg output is sent to stdout, or to a file.

Each point and line is given a name, which can be used to refer to it should you wish to, say, join two points by a line or define a point as the intersection of two lines.

svg files can be opened directly by most browsers as well as by inkscape. They can be rasterized into png by doing something along the lines of

inkscape -y 255 --export-png=picture.png -d 300 picture.svg

4.1 Instructions

4.1.1 Comments

Text between (and including) // and the end of line are comments and are ignored.

4.1.2 Svg Directives

These are instructions to change the rendering of the figure rather than it’s structure. They take the form #svg directive where directive can be one of the following:

4.1.2.1 Line Directives

  • draw
    opposite of nodraw: show following points and lines.

  • nodraw
    don’t actually show following points and lines. They do exist though and can be used to construct further points and lines, whether hidden or not.

  • solid
    draw solid lines. This also sets draw.

  • dashed
    draw dashed lines. This also sets draw.

4.1.2.2 Colour Directives

Colour directives affect the colour of lines only. They aren’t revoked by the line directives, so you can have coloured dashed lines and so on. Colours are strictly limited!

  • colR Red

  • colG Green

  • colB Blue

  • colC Cyan

  • colM Magenta

  • colY Yellow

  • colK blacK

  • colW White

4.1.2.3 Examples

#svg dashed
#svg colM

4.1.3 Points, Lines &c

This is what you’ve all been waiting for. If you want a point, you say pt: for a line, you say line.

The lists below show exactly what to say, words in <angle> brackets indicate where you should supply a value like the name of a line or value of a coordinate.

pointRef and lineRef are the names of previously defined points and names.

4.1.3.1 Variables

Variables can be declared like so:

var variableName 42.0

they can be referenced by name and used in place of x and y coordinates in point definitions.

4.1.3.2 Points

The obvious way is to give (x,y) coordinates of the desired point. They can also be specifed as the intersection of two lines, the midpoint of two points, midpoint of a line or the circumcentre of a triangle.

Point definitions are:

pt <name> <integer>|<varRef> <integer>|<varRef>

defines a point <name> at (x, y)

pt <name> inter <lineRef> <lineRef>

defines a point <name> at the intersection of the lines <lineRef> and <lineRef>

pt <name> bis <pointRef> <pointRef>

defines a point <name> at midway between the lines <lineRef> and <lineRef>

pt <name> bis <lineRef>
defines a point <name> midway between the ends of <lineRef>
pt <name> <pointRef> <pointRef> <pointRef>
defines a point at the intersection of the bisectors of lines point1 - point 2 and point2 - point3, ie the circumcentre.

Examples

pt A 320 160
pt B 400 600
pt C 180 640
pt P bis BC
pt Q bis A B
pt I inter AC BD
pt circumcent A B B'

4.1.3.3 Lines

The simplest way is to define a line from point to point. There are also ways of creating perpendiculars and parallels along with moves and extensions of existing lines.

Line definitions are:

line <name> <pointRef> <pointRef>

defines a line <name> from <pointRef> to <pointRef>

line <name> perp <lineRef> <pointRef>

defines a line <name> perpendicular to <lineRef> through <pointRef>

line <name> para <lineRef> <pointRef>

defines a line <name> parallel to <lineRef> through <pointRef>

line <name> ext <lineRef> <integer>

defines a line <name> which extends <lineRef> by <integer> at each end. If integer is negative, the line contracts.

line <name> sli <lineRef> <integer>

defines a line <name> slides <lineRef> by <integer>, that is, a new line segment is created by sliding <lineRef> along by <integer>. Positive integers cause movement in the positive x direction and vice versa. Vertical lines move up for positive integers.

Examples

line AB A B
line BC B C
line leftPara para AC B
line leftSlide sli leftPara -100
line perpAC perp AC C

4.1.3.4 Circles

Circles can be defined as (centre, radius), (centre, point) or as the circumcentre of a triangle:

circ <name> <pointRef> <integer>.

defines a circle <name> centred at <pointRef> radius <integer>

circ <name> <pointRef> <pointRef>

defines a circle <name> centred at <pointRef> passing through <pointRef>.

circ <name> <pointRef> <pointRef> <pointRef>

defines a circle <name> passing through all three points.

Examples

circ circumc A B B'
circ c50 B' 50
circ c4 A B

4.1.3.5 Arcs

Arcs can be defined as (centre, radius, point, point) or (centre, radius, angle, angle). The definition respects the from - to order the arc is specified in: unlike ang which always choses the non reflex arc.

arc <name> <integer> <pointRef1> <pointRef2> <pointRef3>

defines an arc radius <integer> centred at pointRef1, starting on a line to pointRef2 ending on a line to pointRef3 with the angle measured in a positive sense.

arc <name> <integer> <pointRef1> <integer1> <integer2>

defines an arc radius <integer> centred at pointRef1, starting at angle <integer1> degrees ending at <integer2> degrees, with the angle measured in a positive sense.

Examples

arc arc2 75 D 315 0
arc arc1-internal 50 A D B

4.1.3.6 Angles

Angles can be marked and labelled like so:

ang <name> <pointRef1> <pointRef2> <pointRef3>

draws an arc from <pointRef2> to <pointRef3> centred on <pointRef1> labelled with <name>.

Examples

ang ad A I D
ang cd C I D

4.1.3.7 Identifiers

pointRefs and lineRefs can be made up of letters a-z, A-Z, hyphens, single and double quotes.

Examples

A
A'
A"
A_B
a-b

5 Dodges

If a line segment is on the right line, but in the wrong place, use something like

pt pointA
pt pointB
#svg nodraw
line wrongLine pointA pointB
#svg draw
line rightLine sli wrongLine 42

Forgetting to use #svg nodraw ... #svg draw can lead to confusion and dismay!

If a line segment isn’t quite reaching as far as you’d like then do something like

pt pointA
pt pointB
#svg nodraw
line wrongLine pointA pointB
#svg draw
line rightLine ext wrongLine 250

Identifiers - point and line names - are restricted to ascii characters. Should you need more expressive identifiers unicode escapes can be used, examples follow:

6 Example

Given this input:

// quadrilateral
pt O 0 0

var Ay 160

pt A 320 Ay
pt B 400 600
pt C 180 640
pt D 115 360

#svg colK
line AB A B
line BC B C
line CD C D
line DA D A
#svg colB
line AC A C
line BD B D

#svg dashed
line leftPara para AC B
#svg nodraw
line rightPara para AC D
#svg dashed
line Dext sli rightPara -100

pt P bis BC
pt Q bis D C
pt R bis A C

#svg colM
#svg dashed
line BR B R
line AP A P
line DR D R
line AQ A Q

#svg dashed
pt I inter AC BD
pt X inter BR AP
pt Y inter DR AQ

line perpAC perp AC C

pt B' inter leftPara perpAC
pt D' inter rightPara perpAC

#svg solid
ang &#952; D I A

This command line…

vecCalc.exe -o example.svg example.vecCalc

…produces this figure.

7 Programmer’s Notes

7.1 Programming model

The user supplies a specification of a drawing, in the form of point definitions, line definitions and rendering directives, one per line.

Lines and points must be defined before use. Points have their coordinates resolved on defintion and are entered into a points table. Lines are stored as references to their beginning and ending points.

The users specification is processed into a schedule. This schedule is executed by drawing the points and lines in the order specified, subject to the rendering directives. The rendering directives are put in the schedule as tokens which set flags used when executing the schedule. An include directive immediately inserts the file contents into the output.
